# v7.3.2 관리자 마이페이지/로그아웃 접근 복구 보고서

## 문제
v7.3.1에서 관리자 배지를 `/admin`으로 연결하면서, 관리자 계정이 상단 헤더에서 마이페이지로 바로 이동하기 어려워졌습니다. 그 결과 로그아웃 버튼이 있는 마이페이지 접근성이 떨어졌습니다.

## 반영 내용
1. 상단 헤더 수정
   - 로그인 상태에서 항상 `내 계정` 버튼 표시
   - 관리자 배지는 `/admin`으로 이동
   - `내 계정`은 `/account`로 이동
   - `문제 풀기` 버튼 유지

2. 관리자 대시보드 `/admin` 상단 버튼 추가
   - 마이페이지
   - 오류 제보 관리
   - 로그아웃

3. 오류 제보 관리 `/admin/reports` 상단 버튼 추가
   - 대시보드
   - 마이페이지
   - 로그아웃

4. 로그아웃 동작
   - 관리자 페이지에서 로그아웃 버튼 클릭 시 `signOutService()` 실행 후 `/login`으로 이동

## Supabase SQL
이번 버전은 추가 SQL 실행이 필요 없습니다.

## 기대 효과
관리자가 `/admin` 또는 `/admin/reports`에 들어가도 다시 마이페이지로 이동하거나 바로 로그아웃할 수 있습니다.
