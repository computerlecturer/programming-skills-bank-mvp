# Python 출력형 허용 정답 보강 보고서

## 반영 목적
- Python 리스트, 튜플, 딕셔너리 출력 결과에서 쉼표·콜론 주변 공백 차이 때문에 오답 처리되는 문제를 줄였습니다.
- Python 실제 출력 표기는 대표 정답으로 유지하되, 학습 목적상 의미 차이가 없는 표기 차이는 정답 처리하도록 했습니다.

## 반영 기준
- `['A', 'B', 'C']`와 `['A','B','C']`처럼 쉼표 뒤 공백만 다른 경우 허용
- 문자열 원소의 작은따옴표/큰따옴표 차이 허용
- 딕셔너리의 콜론 뒤 공백 차이 허용
- 여러 줄 출력 중 일부 줄이 리스트/튜플/딕셔너리인 경우 해당 줄의 표기 차이도 허용
- 문자열 내부 공백은 임의로 제거하지 않음

## 수정 방식
1. Python 문제 데이터의 `acceptedAnswers`에 출력형 표기 변형을 추가했습니다.
2. 채점 로직에서 Python 자료구조 출력형에 한해 구분자 주변 공백을 유연하게 비교하도록 보강했습니다.
3. SQL, Java, Linux 채점 기준은 변경하지 않았습니다.

## 수정 문항 수
- Python 30문항

- 116, 119, 121, 122, 123, 124, 128, 129, 130, 142

## 대표 예시

### 36번
대표 정답:
```text
['A', 'B', 'C']
```
추가 허용 정답:
```text
['A','B','C']
["A", "B", "C"]
["A","B","C"]
```

### 66번
대표 정답:
```text
{'B': 2}
```
추가 허용 정답:
```text
{'B':2}
{"B": 2}
{"B":2}
```

### 104번
대표 정답:
```text
[(1, 'A'), (2, 'B')]
```
추가 허용 정답:
```text
[(1,'A'),(2,'B')]
[(1, "A"), (2, "B")]
[(1,"A"),(2,"B")]
```

## 검토 메모
- 숫자 출력, 일반 문자열 출력, SQL/JAVA/Linux 정답 채점 기준은 그대로 유지했습니다.
- Python 자료구조 출력형 정답만 보강했습니다.
- 문자열 내부 공백은 채점에서 제거하지 않으므로, `"A B"`와 같은 값은 그대로 의미를 유지합니다.
