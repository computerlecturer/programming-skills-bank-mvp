# v7.2.9 오류 제보 상세 검토 개선 보고서

## 목적
v7.2.8에서 오류 제보 저장과 관리자 확인 기능이 동작하는 것을 확인했습니다. v7.2.9에서는 관리자가 제보를 보고 실제 수정 여부를 판단할 수 있도록 상세 검토 화면을 강화했습니다.

## 반영 내용
1. 오류 제보 상태값에 `수정 필요` 추가
   - 기존: 접수 / 확인중 / 수정완료 / 반려
   - 변경: 접수 / 확인중 / 수정 필요 / 수정완료 / 반려

2. 관리자 상세 화면에 문제 원문 표시
   - 과목
   - 문항 번호
   - 문제 유형
   - 단원
   - 문제 지문

3. 관리자 상세 화면에 정답/해설 표시
   - 단답/서술형 정답
   - 결과표 정답
   - 문제 해설 HTML

4. 필터 기능 강화
   - 처리 구분: 전체 / 미처리 / 처리완료
   - 상태 필터
   - 과목 필터
   - 오류 유형 필터

5. 운영 흐름 개선
   - `수정 필요` 상태를 통해 실제 JSON 문제 데이터 수정 대상만 따로 관리할 수 있습니다.
   - `수정완료/반려` 제보는 처리완료 영역으로 분리해 볼 수 있습니다.

## Supabase 적용
`supabase/migrations/v7_2_9_question_reports_review_workflow.sql`을 SQL Editor에서 1회 실행해야 `수정 필요` 상태 저장이 가능합니다.

## 권장 운영 흐름
접수 → 확인중 → 수정 필요 → 수정완료

또는

접수 → 확인중 → 반려
