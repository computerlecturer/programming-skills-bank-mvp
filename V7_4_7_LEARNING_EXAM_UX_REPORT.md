## 문제 품질 수정

### Python 59번

기존 문항은 `set`을 정답으로 요구했지만, 지문에 “중복 제거” 또는 “출력 결과 3” 단서가 부족했습니다.

수정 내용:

- 지문에 “중복값을 제거한 뒤, 남은 값의 개수를 출력” 조건 추가
- 해설에 `list`, `tuple`이 아니라 `set`을 써야 하는 이유 보강
- `python:59` 학습자료를 set 중복 제거 중심으로 재작성

### Python 77번

기존 문항은 마지막 값 `C`를 출력하라는 조건만 있어 정답이 `2`와 `-1` 모두 가능했습니다.

수정 내용:

- 지문에 “음수 인덱스” 조건 추가
- 조건 문장에 “양수 인덱스 2가 아니라 뒤에서부터 접근하는 음수 인덱스 사용” 명시
- 해설에 `a[2]`도 같은 출력이지만 문제 의도상 `-1`이 정답인 이유 보강
- `python:77` 학습자료를 음수 인덱스 중심으로 재작성

## 학습모드 개선

### 답 보기 버튼 추가

학습모드에서 학생이 답을 모를 때 바로 정답과 해설을 확인할 수 있도록 `[답 보기]` 버튼을 추가했습니다.

동작 기준:

- 답 보기는 정답 처리하지 않음
- 답 보기를 누른 문제는 `psb:answer-reveals` localStorage에 별도 기록
- 학습 요약에 `답 봄` 개수 표시
- 문제 번호판에서 답을 본 문제는 노란색으로 표시
- 상태 필터에 `답 본 문제` 추가

### 기존 학습하기 유지

`학습하기`는 정답을 바로 공개하지 않는 개념 학습 팝업으로 유지했습니다.

- 학습하기: 힌트·개념·풀이 순서
- 답 보기: 정답·해설 공개
- 정답 확인: 사용자가 입력한 답 채점

## 실전모드 개선

### 순차 풀이 제한

실전모드에서는 현재 문제를 제출해야 다음 문제로 이동할 수 있도록 변경했습니다.

동작 기준:

- 제출 전 다음 문제 버튼 비활성화
- 문제 번호판에서 아직 도달할 수 없는 문제는 잠금 표시
- 이전에 제출한 문제는 다시 이동 가능
- 회원 접근 제한과 순차 풀이 제한을 동시에 적용

### 내가 제출한 답 확인

실전모드에서 제출한 문제는 결과 카드에서 `내가 제출한 답`을 확인할 수 있도록 변경했습니다.

동작 기준:

- 정답인 경우: 정답 여부와 내가 제출한 답 표시
- 오답인 경우: 오답 여부와 내가 제출한 답 표시
- 실전모드 오답에서는 정답과 해설을 바로 공개하지 않음
- 저장된 풀이 기록이 있는 문제를 실전모드에서 다시 열면 기존 제출 답안을 불러옴
