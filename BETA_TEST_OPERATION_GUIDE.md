# 무료 베타 테스트 운영 가이드

본 문서는 프로그래밍기능사 실기 문제은행 MVP를 무료 베타로 운영할 때 확인해야 할 운영 흐름을 정리한 문서입니다.

## 1. 현재 운영 정책

```text
비회원: 과목별 50문항 무료
회원가입 사용자: 베타 기간 전체 620문항 무료
결제 기능: 비활성화
AI 문제 변형: 일반 회원 비활성화
관리자 계정만 AI 테스트 가능
로그인 방식: 이메일 + 비밀번호
문제 데이터: JSON
오류 제보: Supabase DB 저장
오답노트/보관함/학습 진행 상태: 브라우저 localStorage 저장
```

## 2. 베타 테스트 목적

무료 베타의 목적은 결제가 아니라 다음을 검증하는 것입니다.

```text
문제 품질
정답/해설 오류
학생이 헷갈리는 문항
회원가입/로그인 안정성
관리자 오류 제보 처리 흐름
모바일 사용성
재방문 가능성
```

## 3. 학생에게 안내할 문구

베타 안내 문구 예시:

```text
현재 서비스는 무료 베타 테스트 중입니다.
회원가입한 사용자는 베타 기간 동안 전체 문제를 무료로 이용할 수 있습니다.
문제 오류나 개선 의견은 문제 화면의 오류 제보 기능을 통해 접수해 주세요.
AI 문제 변형과 결제 기능은 정식 서비스 전환 시 제공될 예정입니다.
```

## 4. 오류 제보 처리 흐름

학생:

```text
문제 풀이
→ 오른쪽 문제 오류 제보 입력
→ 오류 유형 선택
→ 제보 내용 작성
→ 오류 제보 저장
```

관리자:

```text
/admin 접속
→ 오류 제보 현황 확인
→ /admin/reports 이동
→ 문제 원문/정답/해설 검토
→ 상태 변경
→ 관리자 메모 작성
```

추천 상태 흐름:

```text
접수 → 확인중 → 수정 필요 → 수정완료
접수 → 확인중 → 반려
```

## 5. 문제 수정 운영 흐름

현재 문제 데이터는 JSON에 있습니다.

따라서 관리자 페이지에서 직접 문제를 수정하지 않습니다.

권장 흐름:

```text
오류 제보 확인
→ 수정 필요 상태로 변경
→ JSON 문제 데이터 수정
→ npm.cmd run build
→ 새 버전 ZIP 생성
→ 재배포
→ 수정완료 상태로 변경
```

## 6. 매일 확인할 관리자 지표

관리자 대시보드에서 매일 확인합니다.

```text
전체 오류 제보 수
오늘 접수된 오류 제보 수
미처리 제보 수
수정 필요 제보 수
회원 수
오늘 가입자 수
AI 테스트 사용량
```

## 7. 베타 테스트 참여자 관리

초기 베타는 소수 인원으로 시작하는 것을 권장합니다.

추천 순서:

```text
1차: 내부 테스트 2~3명
2차: 가까운 수강생 5~10명
3차: 실제 학습자 20~30명
4차: 공개 링크 공유
```

각 단계에서 아래를 확인합니다.

```text
회원가입이 되는지
문제 풀이가 끊기지 않는지
오류 제보가 저장되는지
모바일에서 불편한 점이 없는지
학생이 실제로 다시 접속하는지
```

## 8. 유료화 판단 기준

현재는 무료 베타를 유지합니다.

유료화는 아래 조건이 어느 정도 충족된 뒤 검토합니다.

```text
회원 수 증가
반복 방문 발생
오류 제보 감소
문제 품질 안정화
학생이 전체 문제를 실제로 사용
관리자 운영 부담이 감당 가능
```

## 9. 유료화 전 필요한 기능

나중에 유료 전환 전에는 아래 기능을 검토합니다.

```text
비밀번호 재설정
이용약관
개인정보 처리방침
수동 이용권 또는 PG 결제
회원 관리 페이지
결제/이용권 만료일 표시
환불 기준 안내
```

## 10. 현재 보류 중인 기능

```text
문제 데이터 Supabase 이전
학생별 풀이 기록 DB 저장
결제 기능
AI 실제 문제 변형 API 호출
비밀번호 재설정 이메일
관리자 문제 직접 수정 페이지
```

이 기능들은 베타 반응과 운영 필요성이 확인된 뒤 단계적으로 추가합니다.
