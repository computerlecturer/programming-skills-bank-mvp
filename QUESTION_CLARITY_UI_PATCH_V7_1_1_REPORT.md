# v7.1.1 문제 명확성 및 AI 버튼 UI 패치 보고서

## 수정 목적

사용자가 51번 SQL 문항에서 별칭 요구사항이 지문에 드러나지 않아 정답을 한 번에 판단하기 어렵다고 지적하였다. 또한 AI 문제 변형 영역에서 `난이도 높이기` 버튼이 화면 오른쪽에서 잘리는 문제가 확인되어 UI를 보정하였다.

## 문제 지문 수정

SQL 작성형 문항 중 정답 SQL에는 별칭이 포함되어 있으나 지문에서 별칭 요구가 약하거나 빠져 있는 문항을 점검하고, 명확히 해야 하는 문항을 수정하였다.

| 문항 | 수정 내용 |
|---|---|
| SQL 51번 | 사원 수 컬럼 별칭을 `사원수`로 지정하라는 조건 추가 |
| SQL 54번 | 평균 급여 컬럼 별칭을 `평균급여`로 지정하라는 조건 추가 |
| SQL 60번 | 매출 건수 컬럼 별칭을 `건수`로 지정하라는 조건 추가 |
| SQL 150번 | 상사 이름 컬럼 별칭을 `상사이름`으로 지정하라는 조건 추가 |

수정 대상 파일:

- `src/data/sql_200_questions_platform_student_final_reviewed.json`
- `src/data/sql_200_questions_platform_reviewed.json`
- `src/data/study_guides_reviewed.json`

## 정답 허용값 정리

별칭을 지문에서 명시한 문항은 별칭 없는 답안이 정답으로 인정되지 않도록 `acceptedAnswers`를 정리하였다.

- SQL 51번: `COUNT(*) AS 사원수` 필수
- SQL 54번: `AVG(급여) AS 평균급여` 필수
- SQL 60번: `COUNT(*) AS 건수` 필수
- SQL 150번: `상사 이름 AS 상사이름` 필수

## AI 문제 변형 버튼 UI 수정

`src/components/service/ai-variation-panel.tsx`를 수정하여 버튼 영역이 오른쪽으로 넘치지 않도록 변경하였다.

수정 내용:

- 버튼 영역을 고정 폭 flex 구조에서 반응형 grid 구조로 변경
- `현재 난이도 유지`, `난이도 높이기` 버튼이 화면 폭에 따라 1열 또는 2열로 배치되도록 수정
- 버튼 텍스트가 작은 화면에서도 잘리지 않도록 `min-w-0`, `whitespace-normal`, `shrink-0` 적용

## 검증

- SQL 데이터 파일 2종 모두 200문항 유지 확인
- 학습하기 데이터 620개 유지 확인
- SQL 51·54·60·150번 지문/정답/학습하기 연동 수정 확인
- AI 버튼 영역 코드 레벨 반응형 수정 확인

## 비고

이 패치는 v7.1 베타 회원제 정책은 유지하면서 문항 명확성과 UI 잘림 문제만 보완한 v7.1.1 성격의 수정이다.
