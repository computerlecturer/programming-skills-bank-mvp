# SQL 전체 검토 및 반영 보고서

## 반영 항목

1. SELECT * 유형 추가 정답 보강
- 1번, 12번, 32번은 SELECT * 대신 문제에 제시된 전체 필드명을 직접 나열해도 정답 처리되도록 acceptedAnswers를 추가했다.
- 125번 INNER JOIN 전체 열 조회 문제도 전체 필드를 명시한 답안을 추가 정답으로 반영했다.

2. DISTINCT 학습 내용 보강
- 4번 문제 해설에 DISTINCT의 의미와 실행 결과 예시를 추가했다.
- 4번 이론 학습 카드의 제목, 구조, 이론, 접근 방법, 주의사항을 DISTINCT 중복 제거 중심으로 수정했다.

3. 테이블 구조 표 형식화
- SQL 문제 전체에서 [테이블 구조]가 텍스트 형태의 테이블명(필드...)로 보이던 부분을 표 형태(테이블명 / 필드)로 변환했다.

4. 10번 유형 보강
- 거래내역 테이블 구조를 거래번호, 거래일자, 금액 3개 필드가 아니라 거래번호, 거래일자, 금액, 거래구분, 담당자 5개 필드로 확장했다.
- 실제 조회 대상은 거래번호, 거래일자, 금액만 되도록 요구사항을 명확히 했다.
- 문제 지문에 예시 데이터 표를 추가했고, 정답 확인 시 해설에 실행 결과 예시 표가 보이도록 수정했다.

5. 정답 확인 시 실행 결과 예시 추가
- SELECT 작성형 주요 문항(1, 2, 4, 6, 9, 10, 11, 12, 15, 16, 19, 20, 21, 22, 27, 29, 32번)에 실행 결과 예시 표를 해설 영역에 추가했다.

## 운영 메모

- 기존 SQL 전체 검토, LEFT OUTER JOIN 추가 정답, 서술형 객관식 전환 내용은 유지했다.
