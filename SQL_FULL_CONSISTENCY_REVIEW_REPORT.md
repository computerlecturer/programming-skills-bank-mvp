# SQL 전체 검토 및 정합성 수정 보고서

## 검토 기준

SQL 200문항을 다음 기준으로 재점검했다.

1. 요구사항에 명시된 조회 대상과 실제 정답의 SELECT 컬럼이 일치하는지 확인했다.
2. 기준 테이블의 모든 행을 포함하는 LEFT JOIN 문항에서 기준 테이블의 식별 컬럼이 빠져 혼란을 주는지 확인했다.
3. INNER JOIN과 JOIN, LEFT JOIN과 LEFT OUTER JOIN처럼 SQL에서 같은 의미로 쓰이는 표현을 추가 정답으로 반영했다.
4. 결과 작성형 문항은 테이블 행 수, 필드 수, 조건, 정렬, NULL, 중복 여부가 너무 단순하지 않은지 확인했다.

## 주요 수정 사항

### 136번 수정

기존 문항은 회원 테이블을 기준으로 주문번호를 조회한다고 했지만 정답이 회원 이름과 주문번호만 조회하도록 되어 있어, 회원 테이블의 식별 컬럼인 회원번호가 빠져 있었다. 이를 다음과 같이 수정했다.

- 요구사항: 회원번호, 이름, 주문번호 조회로 명확화
- 정답: SELECT 회원.회원번호, 회원.이름, 주문.주문번호 ... 형태로 수정
- 추가 정답: LEFT OUTER JOIN, 별칭 사용 답안 추가

### 145번 수정

136번과 같은 유형의 LEFT JOIN 문항이므로 동일하게 회원번호를 포함하도록 수정했다.

- 요구사항: 회원번호, 이름, 주문번호 조회로 명확화
- 정답: LEFT JOIN / LEFT OUTER JOIN 모두 인정
- 별칭 사용 답안 추가

### 139번 수정

부서 테이블의 모든 부서를 기준으로 조회하는 문항이므로, 부서 식별이 가능하도록 부서코드와 부서명을 함께 조회하도록 수정했다.

- 요구사항: 부서코드, 부서명, 사원 이름 조회로 명확화
- 정답: LEFT JOIN / LEFT OUTER JOIN 모두 인정
- 별칭 사용 답안 추가

### JOIN 표현 추가 정답 반영

다음 문항에는 SQL 표준 사용 관점에서 허용 가능한 대체 표현을 추가했다.

- 121, 123, 125, 126, 129, 132, 133, 193번: INNER JOIN과 JOIN 모두 인정
- 136, 139, 145번: LEFT JOIN과 LEFT OUTER JOIN 모두 인정
- 142, 149, 150번: SELF JOIN 문항에서 LEFT OUTER JOIN 대체 표기 인정

## 이전 데이터 품질 보강 유지

이전 베타 품질 패치에서 보강한 결과 작성형 문항 14개는 유지했다.

- 124, 128, 131, 135, 137, 140, 144, 148, 152, 153, 157, 160, 192, 196번
- 테이블 행을 3~5행 이상으로 확대
- 조건, 정렬, NULL, 중복값을 포함해 단순 암산형 문제가 되지 않도록 조정

## 교사용 검토 의견

이번 수정의 핵심은 “SQL문이 실행 가능한가”보다 “문제 요구사항을 학생이 어떻게 해석할 것인가”에 맞췄다. 특히 LEFT JOIN 문항은 기준 테이블의 모든 행을 포함한다는 조건이 있으면 기준 테이블의 식별 컬럼을 함께 보여주는 편이 교육적으로 더 명확하다.
