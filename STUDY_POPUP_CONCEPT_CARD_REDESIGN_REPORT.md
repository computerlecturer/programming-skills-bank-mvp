# 학습하기 팝업 개념카드형 재설계 보고서

## 적용 목적

기존 학습하기 팝업은 내용은 많지만 긴 문단과 좌우 카드 구조 때문에 학생이 무엇을 먼저 봐야 하는지 파악하기 어려웠다. 이번 수정에서는 학습하기를 정답 해설이 아니라 문제 풀이 전 개념을 정리하는 영역으로 재정의했다.

## 적용 구조

학습하기 팝업을 다음 5단계 구조로 변경했다.

1. 학습하기 사용법
2. 이 문제에서 필요한 개념
3. 예시로 이해하기
4. 문제 풀 때 확인할 순서
5. 자주 하는 실수
6. 조금 더 자세히 보기

## 주요 변경 사항

- 기존 긴 문단 중심 이론 설명을 뒤로 보내고, 상단에는 30초 요약을 배치했다.
- 예시는 실제 문제와 다른 값으로 구성하여 정답이 직접 노출되지 않도록 했다.
- 풀이 순서와 자주 하는 실수를 분리하여 학생이 문제 풀이 전 확인할 항목을 명확히 했다.
- 자세한 이론 설명은 접기/펼치기 형태로 배치해 팝업 첫 화면의 부담을 줄였다.
- Python, Java, Linux, SQL 전체 620개 학습하기 데이터에 quickPoints와 examples 필드를 추가했다.
