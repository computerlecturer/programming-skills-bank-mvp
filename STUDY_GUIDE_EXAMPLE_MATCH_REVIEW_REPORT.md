# 학습하기 예시 매칭 재검토 보고서

## 수정 기준

- 예시는 실제 문제의 코드·값·정답을 그대로 사용하지 않는다.
- 예시는 현재 문제를 풀기 위한 같은 개념을 다른 값으로 보여준다.
- `이 문제에서 필요한 개념`, `예시로 이해하기`, `풀이 순서`, `자주 하는 실수`가 같은 주제를 향하도록 맞춘다.
- 문제 풀이 전 학습자료이므로 정답을 직접 노출하지 않는다.

## 반영 범위

- SQL 200개
- Python 150개
- Java 150개
- Linux 120개
- 총 620개 학습하기 항목의 quickPoints/examples/approach/cautions를 단원 개념 기준으로 재매칭

## 대표 수정 예

### Java 35번
- 기존: 문자열 비교와 print 예시가 들어가 조건문/삼항 연산자 문제와 맞지 않았음
- 수정: `score >= 60 ? "pass" : "fail"`, `if/else`, `a > b ? a : b` 형태의 학습용 예시로 변경

### Java 43번
- 기존: 출력문 예시 중심
- 수정: for문 누적, 반복 변수 출력, while 반복 예시로 변경

## 검토 결과

- JSON 파싱 정상
- 학습 예시는 문제와 동일한 값 대신 다른 샘플 값을 사용
- 학생이 학습하기만 보고 바로 정답을 알 수 없도록 구성
