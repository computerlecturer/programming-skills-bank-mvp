# SQL 200문항 학생 관점 최종 검수 보고서

## 검수 기준

학생이 해당 문항 하나만 보고도 답을 쓸 수 있는지를 기준으로 검토했습니다.

확인 기준은 다음과 같습니다.

1. 테이블명과 컬럼명이 지문에 명확히 제시되어 있는가
2. JOIN / EXISTS / NOT IN / 서브쿼리 문제에서 연결 기준 컬럼이 제시되어 있는가
3. 이전 문항을 참조하는 표현이 없는가
4. result-table 문제에서 정답 구조가 과도하게 노출되지 않는가
5. result-table 문제의 정답표가 실제 SQL 실행 결과와 맞는가
6. 정렬 기준이 애매한 결과표 문제가 없는가
7. 학생이 정답으로 쓸 수 있는 합리적인 대체 답안을 일부 인정하는가
8. 해설이 최소한의 학습 설명을 포함하는가

## 최종 검수 결과 요약

- SQL 총 문항 수: 200문항
- result-table 문항 수: 40문항
- result-table SQL 실행 검증: 40문항 통과
- 이전 문항 참조 표현: 발견되지 않음
- 정답 테이블명 미제시 문제: 발견되지 않음
- result-table 정답 구조 누락: 발견되지 않음

## 이번 최종 검수에서 추가 수정한 문항

### 51번

부서별 사원 수를 구하는 문제에서 정답에는 `AS 사원수` 별칭이 있었지만 지문에서는 열 제목을 `사원수`로 지정하라는 요구가 없었습니다.

수정 내용:
- `AS 사원수`가 없는 답안도 대체 정답으로 인정
- 해설에 별칭은 표시용이고 핵심은 `COUNT(*)`와 `GROUP BY 부서`임을 추가

### 54번

부서별 평균 급여 문제에서 정답에는 `AS 평균급여` 별칭이 있었지만 지문에서는 열 제목을 `평균급여`로 지정하라는 요구가 없었습니다.

수정 내용:
- `AS 평균급여`가 없는 답안도 대체 정답으로 인정
- 해설에 별칭과 HAVING 조건의 역할을 보완

### 52번

`GROUP BY` 결과표 문제에서 정렬 기준이 없으면 학생이 행 순서를 혼동할 수 있었습니다.

수정 내용:
- SQL문에 `ORDER BY 합계 DESC` 추가
- 해설에 합계 내림차순 출력 기준 설명 추가

### 55번

고객별 주문건수 문제에서 `GROUP BY` 결과의 행 순서가 DBMS마다 다를 수 있었습니다.

수정 내용:
- SQL문에 `ORDER BY 고객명 ASC` 추가
- 해설에 고객명 오름차순 출력 기준 설명 추가

### 87번

`DELETE FROM 회원;` 실행 결과를 서술하는 문제는 의미상 맞는 답안 표현이 여러 개 가능했습니다.

수정 내용:
- 다음과 같은 표현을 대체 정답으로 추가
  - 회원 테이블의 모든 데이터가 삭제된다.
  - 회원 테이블의 모든 레코드가 삭제된다.
  - 회원 테이블의 행이 모두 삭제된다.
  - 회원 테이블의 데이터가 모두 삭제된다.
- 해설에 테이블 구조는 남고 행만 삭제된다는 점 추가

### 152번, 153번, 160번, 196번

UNION / UNION ALL 결과표 문제에서 행 순서가 애매할 수 있었습니다.

수정 내용:
- 결과 행 순서를 명확히 하기 위해 `ORDER BY` 추가
- 학생이 결과표 작성 시 행 순서를 덜 헷갈리도록 조정

## 최종 판단

현재 SQL 200문항은 학생이 지문만 보고 풀기 어려운 치명적인 문제는 크게 줄어든 상태입니다.

다만 SQL은 특성상 같은 결과를 내는 답안이 여러 형태로 나올 수 있으므로, 실제 학생 테스트 과정에서 추가 대체 정답이 발견될 수 있습니다. 예를 들어 `NOT IN`과 `NOT EXISTS`, 일부 JOIN/서브쿼리 변형은 같은 결과를 낼 수 있습니다.

따라서 현재 파일은 `학생 테스트용 SQL 1차 최종본`으로 보는 것이 적절합니다.
